On Wed, Oct 23, 2013 at 4:53 PM, Luke Stuyvenberg <stuyvenberg at wisc.edu> wrote:
> I've finished updating the plugin to use (nearly) the latest Micro-Manager base.

Congrats!

> I also expect people using unexpected hardware may receive load errors saying the device adapter interface versions are mismatched. (This may also happen silently, causing unexpected program crashes.)

Do you happen to see any pattern to the undetected incompatibilities
(such as the Micro-Manager version from which the device adapters
came)? (I might want to investigate if (and probably only if) there is
a recurring pattern that looks like more than unrelated bugs or
incompatibilities with non-device-adapter DLLs. Never mind if it
appears unpredictable.)

> N.B.: The latest nightly build has once again changed version numbers, so device adapters from that build will _not_ load in OpenSPIM -- yet. :-)

In case anybody cares, the incompatibility is since October 22 (SVN
r11996) (i.e. MM nightly build device adapters from Oct 22 onward are
incompatible with Luke's present update, or with MM 1.4.15, the latest
release version).

Ironically, that incompatibility was introduced in order to prevent
crashes that could occur due to incompatible device adapters. The
problem was usually seen when using the MultiCamera device, and is
most likely to be observed when using (real) camera device adapters
that are compiled by a different version of Visual Studio.
