Hi,

as I wrote previously, I have the new version of the Picard stage and I  
had issues with the maximum available position being limited to 32 767  
steps, and the slider in the plugin being limited to 9000 um. The solution  
that works for me is:

1) replace PiUSB.dll with the newest version from the Picard website -  
this fixes the 32 767 steps problem
2) I modified the SPIMAcquisition code so that the slider reflects the  
values of Max set during device configuration (where you also set the  
serial numbers). My code is currently here:
https://github.com/openspim/SPIMAcquisition/compare/master...pawlowska:addMaxPropertyReading
I assume that the max value you set during configuration is the maxiumum  
number of STEPS (not um)

maybe this helps somebody.
