Hi Christian,

Sorry for the delay! Windows 7 32-bit shouldn't have much trouble running the OpenSPIM software; we've had several people now run the system on 32-bit Windows 7 with no problems. Is there a particular error you get while running the program on Windows 7? (64-bit OpenSPIM is still being worked on, though some users have suggested that the 32-bit version can run in 64-bit Windows using compatibility mode.)


For changing the axes around, it should work to put in different serial numbers during set-up. When adding the Picard X/Y Stage device, put in to the Serial Number (X) field the number after 'SN' on the X stage sticker. For Serial Number (Y), use the number from the Z stage. Then, when adding the Picard Z Stage device, put in the number from the Y stage. (This is an unusual set-up; if you don't mind my asking, for what kind of experiments are you using the OpenSPIM system? I'm always interested in the unique ways people apply the hardware and software!)


Hope this helps!
Luke
