If you start Micro-Manager from the command-line, you should see some 
output, most likely including an exception. This will be the tell-tale 
what is missing.

Since I tested the procedure on Linux and you are most likely on 
Windows, it is quite likely that there are differences in what you have 
to copy from the `target/dependency/` directory.

> I thought I have to start from Micro-Manager installers instead of

> me another error message:
> 
> 'Failed to load the mmcorej_wrap native library
> mmcorej.MMCoreJJNI.swig_module_init()V'

Plain Fiji does not have Micro-Manager; you will need to install it by 
following the Micro-Manager-dev update site (Help>Update..., then click 
on "Manage update sites", hit the "Add" button and fill in the first and 
second column of the added row with the values "Micro-Manager-dev" and 
"http://sites.imagej.net/Micro-Manager-dev/" respectively; I wrote this 
from memory, some things might be named differently)
