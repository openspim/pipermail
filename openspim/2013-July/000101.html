The plan all along was to get things to a stage where we could contribute
our code to Micro-Manager itself. We have been not very good on that
front, I admit.

In the same spirit as wanting to contribute to Micro-Manager, however, I
have not the faintest problem with merging code into OpenSPIM's source
code that helps you run with a slightly different setup.

As I detailed this morning on the confocal mailing list, the meaning of
"Open" in "OpenSPIM" is intended to imply an inviting project, open to all
interested parties.

Therefore, I encourage you to contribute your patches; I am sure that we
can bring them into a form where the final product works for both Cobolt
and Cube lasers.
