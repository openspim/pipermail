Hi,

I'm not sure if there is a way to get around it. I also had a similar  
error. That is, one way is to use the Multidimensional acquisition of  
Micro-manager itself, although then you can't use rotation.

BTW, my Micro-manager doesn't save files larger than 4GB as well, instead  
it splits the stack in one with 3.99 GB size and another one with the  
rest. So the long-term solution is to change the code of the plugin to  
either do the same - slipt the file - or by adding the  "setBigTiff(true)"  
line (if it really works).
