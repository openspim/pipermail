> Okay, good to know we've at least narrowed down what's going on. In this
> case I suspect the actual problem isn't the version of Micro-Manager, but
> the version of the OME library in Fiji. Sometimes Micro-Manager and Fiji
> depend on the same library, but require different versions of that  
> library,
> which is unfortunate; we're trying to resolve such issues when we  
> discover
> them.
>
> In the meantime, you can try replacing the library manually. This is a  
> jar
> file called "ome-xml.jar", which should be in the "plugins/Micro-Manager"
> directory of the nightly download. You can try just replacing the
> corresponding .jar file in your Fiji installation with the one from the
> nightly build. Make a backup of the original .jar file though, just in  
> case
> something else in Fiji breaks. In the meantime, we can look into what it
> would take to get the update site updated.

The conflict is in other JARs. We still use an outdated version of
scifio.jar, which conflicts with the newer version in Fiji. We fully
intend to fix this, and it will hopefully not be too hard, but it is not
trivial (we need to make sure we don't introduce issues into file saving
in our official release, and that will require careful testing).

Please consider the Micro-Manager-dev update site to be experimental.

I don't think there is an easy fix at this moment that can be done by
just replacing JARs. We'll keep working on it!
