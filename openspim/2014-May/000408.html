Hi Zuomin Zhao,

On 04/30/14, Zuomin Zhao  wrote:
> We used a stradus laser in OpenSPIM. But we cannot open the Acquire
OpenSPIM option from Micro-manager. The settings can be seen in attachments.
It always shows error like "Error in device VLTStradus: Duplicate property
names are not allowed"

Since the Stradus device adapter is "secret", my ability to help is limited.
My suggestion is:
- First, please make sure you are using the Stradus device adapter
(mmgr_dal_Stradus.dll) from the 5 February 2014 nightly build of
Micro-Manager[1].
- Run OpenSPIM, but before clicking "Acquire SPIM Image", create a pixel
size:
 - On Micro-Manager, click "Tools -> Pixel Size Calibration..."
 - Click "New"
 - Set Label to "Uncalibrated"
 - Set Pixel Size (um/pixel) to 1.00
 - Across from "Core - Initialize" click "Use" checkbox.
 - Click "OK".
- Try "Acquire SPIM Image" again.


> Another problem is the information in Corelog20140430 that "Startup script
file (C:\Users\Admin\Desktop\Fiji.app/MMStartup.bsh) not present." I do not
know why miss this file MMSartup.bsh. How handle it?
This error should not cause any problems right now. It only means that the
"Acquire SPIM Image" window will not open automatically.


If you continue to have trouble, please e-mail us again and I will try to
help more.
Luke Stuyvenberg
