Hi!

 > This helps a lot and gives many aspects to think about. What is the
 > operating software you are using? µManager or something else, and how
 > well these lasers integrate with the software?

We use our own software QuickFit 3 
(http://www.dkfz.de/Macromol/quickfit/), which is kind of optimized for 
our setup and application (SPIM-FCS). There are drivers for the CObolt 
lasers, and Andor EMCCD cameras (NOT for the sCMOS, as they use the SDK 
3), but new drivers can always be written in C++. The whole application 
is multi-platform, i.e. we run our instrument on Linux and the data eval 
typically on windows. It's written in C++ and uses Qt 4.8 for the GUI.

But I think that using µManager may probably be easier for you ... also 
the community is larger ;-)

One more thing on the beam expanders: As first beam expanders, we use 
commercial units (one is a zoom expander 1x-8x from sill optics, the 
other one is a fixed 5x from Qioptic/Linos), which are both NOT 
achromatically corrected. So this might be part of our problem ... but 
then having the possibility to tune both lasers independently might be 
worth thinking about. At least for our application (single-cell imaging 
with a thin lightsheet) it helps a lot!

Best,
JAN
